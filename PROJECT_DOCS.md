# 家味 - 家庭菜谱管理器

## 项目概述

家味是一款家庭菜谱管理应用，旨在帮助用户便捷地记录、查看和管理家庭日常菜谱。
目标用户: 家庭烹饪爱好者、需要管理日常饮食的用户。
核心问题: 解决家庭菜谱分散、难以系统化管理的问题。

## 技术选择

### 核心框架和技术栈
- **React**: v19.2.3 - 用于构建用户界面的JavaScript库
- **TypeScript**: ~5.8.2 - 为JavaScript添加静态类型的编程语言
- **Vite**: v6.2.0 - 下一代前端构建工具，提供更快的开发体验
- **Node.js**: 运行环境（建议v18+）

### 开发依赖
- **@types/node**: v^22.14.0 - Node.js的类型定义
- **@vitejs/plugin-react**: v^5.0.0 - 为Vite提供React支持
- **vite**: v^6.2.0 - 构建工具

### 项目配置
- **TypeScript配置**:
  - 目标: ES2022
  - 模块系统: ESNext
  - JSX转换: react-jsx
  - 模块解析: bundler
  - 别名: @/* -> ./*

- **Vite配置**:
  - 开发服务器端口: 3000
  - 开发服务器主机: 0.0.0.0

## 各文件功能说明

### 根目录文件

#### `.env.local`
- **功能**: 可选的环境变量配置文件
- **重要性**: 如需扩展功能可使用

#### `App.tsx`
- **功能**: 应用程序的主要组件，负责路由和状态管理
- **职责**:
  - 管理页面状态（主页、添加菜谱、菜谱详情、购物车）
  - 维护菜谱列表和购物车数据
  - 处理页面导航和路由
  - 管理全局分类

#### `index.html`
- **功能**: 应用程序的主HTML模板文件
- **职责**: 提供React应用挂载点和基本HTML结构

#### `index.tsx`
- **功能**: 应用程序的启动入口
- **职责**: 渲染App组件并将其挂载到DOM元素上

#### `package.json`
- **功能**: 项目配置文件，定义了项目依赖、脚本和其他元数据
- **内容**:
  - 项目名称、版本号
  - 依赖项（React、React DOM）
  - 开发依赖项（TypeScript、Vite等）
  - 构建脚本（dev、build、preview）

#### `tsconfig.json`
- **功能**: TypeScript编译器配置文件
- **职责**: 定义TypeScript如何编译源代码

#### `vite.config.ts`
- **功能**: Vite构建工具配置文件
- **职责**:
  - 配置开发服务器
  - 设置插件（React插件）
  - 配置路径别名

#### `types.ts`
- **功能**: 定义项目中使用的自定义类型
- **内容**: 包括Recipe、CartItem、Page等接口定义

#### `constants.tsx`
- **功能**: 存储应用程序的常量和初始数据
- **内容**:
  - 初始菜谱数据(INITIAL_RECIPES)
  - 分类组(CATEGORY_GROUPS)

#### `metadata.json`
- **功能**: 项目元数据（功能未明确）

#### `README.md`
- **功能**: 项目简介文档

### Pages目录文件

#### `Home.tsx`
- **功能**: 应用程序主页，展示菜谱列表和分类
- **职责**:
  - 显示菜谱网格
  - 提供分类筛选功能
  - 支持拖拽重新排序分类
  - 导航到其他页面

#### `AddRecipe.tsx`
- **功能**: 用于添加新菜谱的表单页面
- **职责**:
  - 提供菜谱输入表单
  - 图片上传功能
  - 分类选择
  - 保存新菜谱

#### `RecipeDetail.tsx`
- **功能**: 显示单个菜谱的详细信息
- **职责**:
  - 展示菜谱详细信息（图片、标题、描述、成分、步骤等）
  - 提供删除功能
  - 返回导航

#### `Cart.tsx`
- **功能**: 购物车页面，管理需要购买的食材
- **职责**:
  - 显示购物车项目
  - 管理项目数量
  - 清空购物车功能
  - 计算总价

## 运行项目

### 开发环境
1. 安装依赖: `npm install`
2. 启动开发服务器: `npm run dev`

### 构建和预览
- 构建生产版本: `npm run build`
- 预览生产版本: `npm run preview`
